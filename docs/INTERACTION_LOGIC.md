# Логика взаимодействия Opticanew

Этот документ фиксирует поведение интерфейса:
- что происходит после каждого нажатия
- как система отвечает пользователю
- как показываются ошибки
- как выглядит success / empty / loading / retry

## Главное правило

Ни одно действие пользователя не должно оставаться без ответа.

Если человек:
- нажал кнопку
- выбрал режим
- отправил пустое поле
- получил ошибку

интерфейс обязан явно показать, что произошло.

---

## 1. Состояния любого действия

У каждого важного действия должны быть состояния:

1. idle — обычное
2. loading — идёт обработка
3. success — результат готов
4. empty — данных недостаточно
5. error — произошла ошибка
6. retry — можно попробовать ещё раз

---

## 2. Кнопки

### Главное правило
Кнопка не должна молчать.

### Для кнопки `Проверить текст`

#### Обычное состояние
Текст кнопки:
- Проверить текст

#### Во время обработки
Текст кнопки:
- Анализируем…

Поведение:
- повторное нажатие временно блокируется
- есть визуальный отклик

#### После успеха
- результат появляется сразу под полем
- линза обновляет состояние

#### После ошибки
- появляется понятное сообщение
- кнопка меняется на `Попробовать ещё раз`

---

## 3. Поле ввода

### Если поле пустое
При нажатии на кнопку система должна:
- вернуть фокус в поле
- показать сообщение рядом с полем

Текст:
`Вставь текст, чтобы линза показала скрытое давление.`

### Если текст слишком короткий
Текст:
`Нужно чуть больше текста, чтобы увидеть приёмы.`

### Если текст корректный
- запускается анализ
- поле остаётся видимым
- результат появляется ниже

---

## 4. Ошибки

### Главное правило
Ошибка никогда не показывается только цветом.

Ошибка всегда должна иметь:
- текст
- визуальное состояние
- понятный следующий шаг

### Допустимые ошибки

#### Ошибка загрузки правил
Текст:
`Не удалось загрузить правила анализа.`

Действия:
- Попробовать ещё раз
- Вернуться на главную

#### Ошибка обработки
Текст:
`Что-то пошло не так. Попробуй ещё раз.`

Действия:
- Повторить

#### Пустой ввод
Текст:
`Вставь текст для анализа.`

---

## 5. Результаты анализа

### Если найдено давление
Показываем:
- уровень давления
- найденные приёмы
- короткое объяснение
- следующий шаг

Следующий шаг:
- Открыть разбор

### Если ничего не найдено
Показываем:
- Сильных триггеров не видно
- Это не значит, что текст идеален

Следующий шаг:
- Проверить другой пример

---

## 6. Логика состояний линзы

Линза тоже отвечает на действия пользователя.

### idle
- спокойная
- тёмная
- мягкий свет

### loading
- центр немного ярче
- оси читаются сильнее
- появляется ощущение фокусировки

### low pressure
- линза спокойная
- нейтральная карточка

### medium pressure
- тёплый ореол
- чуть более выраженный внутренний свет

### high pressure
- контур теплее
- больше напряжения в центре
- без крикливой тревоги

---

## 7. Поведение после нажатия

### Пользователь нажал `Проверить текст`
Система должна:
1. проверить, не пустое ли поле
2. если пустое — показать ошибку рядом с полем
3. если не пустое — показать loading
4. после обработки — показать success или neutral result
5. дать понятный следующий шаг

---

## 8. Следующий шаг после любого состояния

После каждого состояния у пользователя должен быть понятный маршрут.

### После success
- открыть разбор

### После neutral
- проверить другой пример

### После error
- попробовать ещё раз

### После empty
- вставить текст

---

## 9. Мобильное правило

На телефоне особенно важно:
- большие зоны нажатия
- понятный фокус
- короткие сообщения
- ошибки рядом с местом действия
- не ломать экран длинными алертами

---

## 10. Короткая формула

Opticanew не оставляет нажатие без ответа.

Каждое действие:
- получает отклик
- объясняется
- ведёт дальше
