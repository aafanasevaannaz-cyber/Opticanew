# Архитектура Opticanew

## Цель

Сделать один сайт, который:

- понятно работает с телефона
- легко меняется
- выглядит собранно и дорого
- не ломается от правок
- соединяет инструмент и содержание

## Не делаем

- длинную главную страницу с хаотичными секциями
- один огромный HTML-файл со всем сразу
- смешивание текстов, логики и стилей в одном месте

## Делаем

Один сайт, внутри которого есть три уровня.

### 1. Главная
Задача:
- быстро объяснить, что это
- дать первый вход

Пользователь должен понять за 2 секунды:
- это про скрытое давление в тексте
- здесь можно проверить фразу
- здесь можно пойти глубже

### 2. Детектор
Задача:
- дать полезное действие
- показать, где именно текст давит

Показывает:
- уровень давления
- найденные приёмы
- краткое объяснение
- более честную формулировку

### 3. Досье / архив
Задача:
- учить замечать механизмы быстрее
- объяснять логику давления глубже

## Страницы

```text
index.html       -> вход
 detector.html   -> инструмент
 dossier.html    -> одно досье
 archive.html    -> каталог механизмов
```

## Первый экран

Первый экран не должен пытаться рассказать всё.
Он должен делать только одно:

- направлять в первое действие

### На первом экране должно быть:
- бренд
- короткий смысл
- линза как главный визуальный объект
- 1 главный CTA
- 1 вторичный CTA

### На первом экране не должно быть:
- трёхэтажной сетки
- длинных объяснений
- кучи одинаково важных блоков
- визуального шума

## Главная визуальная идея

Opticanew должен ощущаться как:

- тихий дорогой интерьер
- библиотека / архив
- стеклянная исследовательская среда
- линза, которая показывает скрытый слой

## Архитектурный принцип

Сложность раскрывается по слоям:

- экран
- следующий уровень
- улика
- связанный ход

Это значит:
- на первом уровне только самое важное
- аналитика и детали — по явному действию

## Главное техническое правило

Любая часто меняющаяся вещь должна храниться отдельно от разметки.

Это касается:
- текстов
- правил детектора
- досье
- карточек
- служебных подписей
