# Дизайн-система Opticanew

Этот документ фиксирует визуальные правила проекта.

Главная цель:
- сохранить красоту без хаоса
- сделать интерфейс дорогим по ощущению
- удержать читаемость и спокойствие
- не допускать случайных решений от экрана к экрану

---

## 1. Визуальный принцип

Opticanew — это не техно-лендинг, не блог и не киберпанк.

Проект должен ощущаться как:
- тихий исследовательский кабинет
- тёмный дорогой интерьер
- оптический прибор и архив
- стеклянная глубина без декоративной истерики

Короткая формула:

**дорогой тихий прибор, а не шумный сайт**

---

## 2. Главные качества интерфейса

Каждый экран должен быть:
- спокойным
- собранным
- читаемым
- мобильным
- тактильным
- ясным

Интерфейс не должен быть:
- крикливым
- перегруженным
- кислотным
- агрессивно технологичным
- декоративным ради декора

---

## 3. Палитра

### Основа
- глубокий тёплый почти-чёрный фон
- молочно-кремовый текст
- янтарно-медный акцент
- слабый холодный дымчатый оттенок внутри оптических объектов
- приглушённые статусы вместо резких тревожных цветов

### Цветовые роли
- `bg-1` — верхний глубокий тёплый фон
- `bg-2` — нижний почти-чёрный фон
- `text` — основной текст
- `text-soft` — вторичный текст
- `text-faint` — служебный текст
- `accent` — янтарный акцент
- `warn` — мягкая тёплая тревога
- `danger` — приглушённая опасность
- `ok` — спокойная нейтральность

### Запреты
Нельзя:
- использовать кислотные акценты
- делать чисто белые большие поверхности
- строить экран на красных тревожных пятнах
- перегружать интерфейс несколькими акцентами сразу

---

## 4. Типографика

### Принцип
Типографика делится на два слоя:

1. Выразительный смысловой слой
2. Точный интерфейсный слой

### Шрифты
- Display serif — только для больших смысловых заголовков
- Clean sans — для всего интерфейса, форм, кнопок, результатов, архивов

### Правила
- заголовки должны быть крупными, но не разваливать экран
- текст не должен становиться визуальной массой
- длина строк должна быть короткой и спокойной
- большие героические тексты допустимы только там, где нет конкуренции с кучей блоков

---

## 5. Пространство и ритм

### Принцип
Меньше блоков — больше воздуха.

Каждый блок должен ощущаться как предмет, а не как случайная коробка.

### Правила
- не лепить много равнозначных секций
- между крупными блоками должен быть воздух
- на первом экране не должно быть ощущения плотной свалки
- один экран = один главный фокус

---

## 6. Компоненты

### Базовые компоненты
- shell
- topbar
- hero
- lens-card
- floating result card
- toolbox
- pills
- CTA
- ghost button
- result block
- mini-card

### Принцип компонентов
Компоненты должны быть повторяемыми.

Если нужен новый экран, он собирается из уже существующих компонентов,
а не рисуется заново с новым стилем.

---

## 7. Кнопки

### Главная кнопка
Главная CTA должна ощущаться как управляющий элемент прибора:
- тёплая
- собранная
- уверенная
- без дешёвого маркетингового блеска

### Вторичная кнопка
Вторичная кнопка должна быть:
- тише
- тоньше
- не спорить с главным CTA

### Запреты
- две одинаково кричащие кнопки рядом
- агрессивные тени
- лишняя псевдо-объёмность

---

## 8. Карточки

Карточки в Opticanew — это не рекламные плитки.

Они должны ощущаться как:
- таблички
- досье-панели
- кассеты с уликами
- исследовательские контейнеры

### Правила
- крупные радиусы
- тонкие линии
- мягкая глубина
- без визуального шума

---

## 9. Статусы

Состояния интерфейса не должны быть кричащими.

### low / neutral
- спокойный зеленовато-дымчатый тон

### medium / warning
- мягкий янтарный

### high / danger
- приглушённый терракотово-ржавый

### Главное правило
Статус должен объяснять, а не паниковать.

---

## 10. Анимация

Анимация допустима только как подтверждение жизни интерфейса.

### Допустимо
- мягкий отклик кнопки
- небольшая смена света в линзе
- плавное появление результата
- деликатное усиление осей при анализе

### Недопустимо
- резкие вспышки
- быстрые перелёты
- тяжёлые модальные прыжки
- постоянное движение ради эффекта

---

## 11. Первый экран

Первый экран должен быть:
- без обязательной прокрутки
- с одной главной линзой
- с одним главным CTA
- с одним вторичным CTA

Если на первом экране начинают спорить:
- линза
- заголовок
- карточки
- архив
- несколько входов

значит экран собран неправильно.

---

## 12. Проверка любого нового экрана

Перед тем как принимать экран, нужно спросить:

- есть ли здесь один главный фокус?
- выглядит ли это как часть Opticanew, а не как новый чужой сайт?
- спокойно ли это читается на телефоне?
- не стало ли слишком много важного одновременно?
- можно ли убрать что-то ещё?

Если ответ плохой — экран упрощается.

---

## 13. Короткая формула

Дизайн Opticanew:
- тихий
- дорогой
- мобильный
- оптический
- объясняющий
- не шумный
